# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Development Commands

- **Update gallery**: After adding or removing photos in `gallery/images/`, run:
  ```bash
  python3 update_gallery.py
  ```
  This scans the images directory and regenerates `gallery/gallery.js` which the website uses to display photos.

- **Local development**: To serve the site locally (recommended for testing photo loading):
  ```bash
  python3 -m http.server 8000
  ```
  Then visit `http://localhost:8000` in your browser.

- **View directly**: You can also open `index.html` directly in a browser, but some browsers may block image loading when opening HTML files from the filesystem.

## Project Structure

- `index.html`: Contains all pages (Home, About, Services, Clients, Our Work, Contact) as hidden sections that are shown/hidden via JavaScript navigation.
- `assets/`: Contains the logo (expected as `logo.png`; a fallback SVG is used if missing).
- `gallery/`: 
  - `gallery.js`: Auto-generated file containing photo metadata (do not edit manually).
  - `images/`: Directory containing photo categories as subfolders (e.g., `kitchens/`, `ceilings/`). Each subfolder name becomes a filter category on the site.
- `update_gallery.py`: Python script that generates `gallery/gallery.js` by scanning `gallery/images/`.

## Architecture

This is a static website built with HTML, CSS, and vanilla JavaScript. Key characteristics:

- **Tech stack**: HTML5, CSS3, ES6 JavaScript. No external libraries or frameworks.
- **Single Page Application (SPA) behavior**: All content is in `index.html`; JavaScript handles navigation between pages by showing/hiding sections.
- **Dynamic gallery**: The photo gallery is populated by reading `window.GALLERY` from `gallery/gallery.js`, which is generated by `update_gallery.py`.
- **CSS custom properties**: Colors, fonts, and spacing are defined as CSS variables in `:root` for easy theming.
- **Responsive design**: Uses CSS Grid and Flexbox with media queries for different screen sizes.
- **Lightbox**: Clicking a gallery image opens a fullscreen lightbox with navigation.
- **Intersection Observer**: Used for scroll-based animations and counting animations.

## Deployment

The entire folder contains only static files and can be deployed to any static hosting service (Netlify, GitHub Pages, Vercel, etc.). Remember to run `python3 update_gallery.py` before deploying whenever photos are added or removed.

## File Notes

- Do not edit `gallery/gallery.js` directly — it is overwritten by `update_gallery.py`.
- The `logo.jpeg` referenced in the HTML is optional; if missing, an SVG fallback logo is used.
- The commented-out second office section in the Contact page can be enabled by removing the HTML comments if needed.
- There are no build steps, dependencies, or package managers required — only Python 3 is needed for the gallery update script and local server.
